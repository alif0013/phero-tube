Question 1: Discuss the scope of var, let, and const :-

Ans: JavaScript, like a lot of more modern languages, provides a host of different methods for variable declaration. JavaScript has the var, let and const keywords for variable declarations and with each comes a different use case. This article aims to dive in-depth into the caveats and nuances which you might encounter when using either of these variable declaration methods.

1.Var : keyword declares a variable which is scoped to its current execution context, optionally initializing to a value.

2.Let : keyword declares a block scoped variable, optionally initializing it to a value..

3.Const : keyword declares constants which are block scoped much like variables defined using let but the value of a constant cannot change. The const declaration creates a read-only reference to a value.

Conclution : var has function or global scope, is hoisted, and can be re-declared. let and const have block scope, are hoisted with a Temporal Dead Zone, and cannot be re-declared in the same scope. const additionally enforces immutability for the assigned value, but it doesn't mean the content of objects or arrays can't be changed. It's recommended to use let and const over var due to their more predictable behavior and better scoping.

Question 2: Tell us the use cases of null and undefined :-

Ans: Null and undefined are both special values in programming languages, often used to indicate the absence or lack of a meaningful value.

1. Undefined : Undefined means the variable has been declared, but its value has not been assigned.

A variable that has not been assigned a value is of type undefined . A method or statement also returns undefined if the variable that is being evaluated does not have an assigned value. A function returns undefined if a value was not returned .

2. Null : Null means an empty value or a blank value.

Null is an object in JavaScript and represents primitive data types. A null value in JavaScript is used for referring absence of any object value and if any function or variable returns null, then we can infer that the object could not be created.

Conclution : undefined is typically used to represent variables that have been declared but not assigned a value, function parameters without provided arguments, and missing object properties. Null, on the other hand, is used to intentionally indicate that a variable or property has no value or to clear an existing value. Both values are important to handle various scenarios where the absence of a value needs to be represented.

Question 3: What do you mean by REST API?:-

Ans: A RESTful API is an architectural style for an application program interface (API) that uses HTTP requests to access and use data.

A REST API (also called a “RESTful” API) is a specific type of API that follows these guidelines. REST stands for Representational State Transfer. This means that when a client requests a resource using a REST API, the server transfers back the current state of the resource in a standardized representation.
